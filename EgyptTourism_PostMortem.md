# تقرير تحليل أعطال بناء تطبيق Egypt Tourism (Post-Mortem)

## 1. الحالة الأصلية (قبل البدء)
* **الإصدارات:** كان التطبيق يستخدم إصدارات "مستقبلية" وغير مستقرة بالخطأ:
    * `React Native: 0.81.5` (إصدار ليلي/Nightly)
    * `React: 19.1.0` (إصدار تجريبي)
    * `Expo: 54.0.33` (إصدار غير رسمي بعد)
* **المشكلة:** كراش فوري عند التشغيل (Native Crash) بدون ظهور أي سجلات (Logs).

---

## 2. ملخص المحاولات (52 محاولة)

### المرحلة الأولى: محاولة تثبيت الوضع الحالي
* **ما تم عمله:** تقييد الـ SDK ليكون 34، تعطيل المعمارية الجديدة (New Architecture)، وتعطيل محرك Hermes.
* **النتيجة:** استمرار الكراش، مما أكد أن المشكلة في "نواة" الإصدارات المستخدمة وتعارضها مع بيئة أندرويد.

### المرحلة الثانية: وضع التشخيص (Diagnostic Mode)
* **ما تم عمله:** تبسيط `App.js` ليعرض فقط جملة "Hello World" وحذف كل المكتبات (Maps, Navigation).
* **النتيجة:** حتى مع "أبسط تطبيق ممكن"، استمر الكراش، وهذا أثبت أن العطل **Native** بنسبة 100% وليس له علاقة بكود الـ Javascript.

### المرحلة الثالثة: التراجع للإصدارات المستقرة (The Great Downgrade)
* **ما تم عمله:** العودة لإصدارات Expo 51 و React Native 0.74.5 (الأكثر استقراراً حالياً).
* **الإجراءات التقنية:** 
    1. حذف `node_modules` و `package-lock.json`.
    2. عمل `npx expo prebuild` لتوليد مجلد `android` جديد كلياً.
* **النتيجة:** ظهرت مشكلة جديدة في الـ Gradle وهي عدم التعرف على موديول `:expo`.

### المرحلة الرابعة: صراع الـ Gradle و SDK Versions
* **المشكلة العنيدة:** ظهور خطأ `compileSdkVersion is not specified` لموديول `:expo`.
* **ما جربناه لحله:**
    * وضع القيم في `gradle.properties`.
    * وضع القيم في `build.gradle` (الرئيسي والفرعي).
    * تعديل ملفات الـ `node_modules` يدوياً لحقن الإصدار 34.
    * استخدام "المسارات المطلقة" (Absolute Paths) في `settings.gradle`.
* **النتيجة:** فشل الـ Gradle في قراءة هذه القيم، وهو سلوك "شاذ" تقنياً.

## 3. الأسباب الجذرية المحتملة (The Killers)
بعد التحليل، هذه هي الأسباب الثلاثة التي منعتنا من النجاح:
1. **Node.js 22:** أنت تستخدم إصدار Node حديث جداً (v22)، وتبين أن سكربتات Autolinking الخاصة بـ Expo 51 **تنهار (Crash)** مع هذا الإصدار (TypeError: path must be string).
2. **Java JVM Mismatch:** الجهاز يستخدم Java 8 افتراضياً بينما المشروع يتطلب Java 17، وتغيير المسار يدوياً في Gradle لم يكن كافياً لبعض موديولات Expo.
3. **Gradle Cache:** يبدو أن نظام الـ Cache في ويندوز كان يمنع الـ Gradle من رؤية التعديلات اليدوية التي كنا نقوم بها داخل الـ `node_modules`.

## 4. خارطة الطريق للعودة (Next Steps)
عند البدء مجدداً، يجب اتباع هذه الخطوات بالترتيب لضمان النجاح:

1. **تعديل البيئة (Environment Fix):**
   * تنزيل إصدار Node.js ليكون **LTS (v20)** بدلاً من v22.
   * تثبيت **Java JDK 17** وجعلها الـ Default في الويندوز.
2. **تنظيف شامل (Hard Reset):**
   * مسح مجلدات `node_modules` و `android` و `.expo` يدوياً من الـ Explorer.
3. **البناء النظيف:**
   * تشغيل `npm install`.
   * تشغيل `npx expo prebuild` (سيقوم الآن بالربط بشكل سليم لأن Node v20 مستقر).
4. **التشغيل:**
   * فتح Android Studio وتشغيل الـ Logcat أولاً قبل الضغط على زر تشغيل التطبيق.
